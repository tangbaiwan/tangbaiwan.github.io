0x00 前言

对于从Unity转到UE4的程序员来说，不得不面临着接触C++这一NB的语言，尽管说语言都有相似之处，但是还是存在不少的不同点。 在C#中我们一般只考虑引用，也只用引用这个词来表述几个对象之间的相互关系。但是在C++中你可能要更多的去使用指针，当然有时候用引用还是相当方便的。在C#中你可能使用readonly来表达一个变量时只读的，但是在C++中你可能需要使用const来表述只读这一行为。 所以区别还是很多的，在此笔者总结了一下C++中一些核心的知识点，方便后续Unity转UE4程序员浏览和查看，同时也方便了自己。废话不多说，开始记录。由于码字仓促，文章内容难免有错误纰漏之处，如若读者能不吝告知，将及时修正。

0x01 基础知识

命名空间

  • 用双冒号作用域表示 ::，如果前面什么也不加，代表全局作用域
  • 作用是为了解决名称冲突
  • 命名空间必须要声明在全局作用域
  • 命名空间可以嵌套。
  • 命名空间是开放的，可以随时将新成员添加到命名空间下。

运算符重载

  • 起因：一方面是因为对于内置的数据类型，编译器知道如何解释处理，但对于自定义的数据类型编译器并不知道；另一方面可以给符号以新的含义。
  • 关键字 operator，例如加号运算符写作 operator+。
  • 实现运算符重载可以有两种实现方式。成员函数 和全局函数内都可以实现运算符重载。 其实本质就是一个函数调用，operator+局部成员函数重载调用的是 p1.operator+(p2)，如果是全局重载了operator+则相当于是调用了operator+(p1,p2)。只是编译器为我们做了这些事而已。
  • 运算符不能滥用，否则会让人混淆，除非有需求。
  • 内置的数据类型运算符不能重载。
  • 两个重要的运算符重载：指针运算符和赋值运算符。 指针运算符重载可以实现智能指针，以托管new出来的对象； 赋值运算符重载则是为了解决浅拷贝，深拷贝的问题。编译器默认为我们提供的operator=其实是值拷贝(浅拷贝)，如果一旦在类中有属性创建在了堆区，那么在释放的时候就会出现内存重复释放的问题，需要重载赋值运算符来解决，改为深拷贝。
  • 关系运算符重载：对于编译器不知道如何进行比较，所以需要重载 ==和 ！=
  • 函数调用运算符重载：重载()，使用起来很方便，仿佛就像函数调用一样，称为仿函数
  • 最好不要重载&&和||两个运算符，重载后将无法保留默认的短路特性。

函数重载

  • 同一个作用域
  • 函数名称相同
  • 函数参数个数，类型，顺序不同
  • 函数的返回值不能作为重载条件

内联函数

  • 产生原因是因为宏存在缺陷，因为宏执行的时候是完全是复制粘贴，所以经常会出现运算和我们预期不符的结果，而函数则不会。
  • 关键字 inline，函数的声明和实现需要都加上才可以。
  • 类中的成员函数默认都加了inline。
  • 内联函数只是给编译器的一个建议而已，编译器并不一定完全遵从执行。例如一些判断，循环，函数体过于庞大这些即便加了inline，编译器也不会按内联执行。有些时候即便没加，例如一些小的函数，编译器在优化的时候也会主动帮我们加上，所以仅仅是对编译器的一个建议。

静态成员

  • 分为静态成员变量和静态成员函数
  • 对于静态成员变量，所有对象共享一份数据；编译阶段就分配内存；类内声明，类外初始化；既可以通过对象访问，也可以通过类名访问(::)
  • 对于静态成员函数，所有对象都共享同一份函数；静态成员函数只能访问静态成员变量，不能访问非静态成员变量；静态成员函数也是存在访问权限的；访问方式依然两种，通过对象访问，通过类名作用域访问(::)

常对象和常函数

  • 关键字 const
  • 对于常对象来说，写法const Person p；不许修改成员属性；只能调用常函数；对象成员不可以用static和const同时修饰。
  • 对于常函数而言，写法 void show() const；常函数本质是为了修饰成员函数中的this指针，让指针指向的值不可以被修改；对于某些特殊的属性，确实需要修改的，可以用multable关键字修改。

友元函数

  • 全局函数作为某个类的友元函数，访问类中的私有成员
  • 可以将某个类作为另一个类的友元类，访问另一个类的私有成员。
  • 也可以将某个类的成员函数作为友元函数，访问另一个类的私有成员。

默认参数

  • 可以给一个函数的形参添加默认值。int func(int a = 10)
  • 默认参数不可以交叉存在。 即如果一个位置有了默认参数，那么从真个位置开始从左到右都必须有默认值。
  • 函数的声明和实现只能有一个提供默认值，不可以同时提供
  • 占位参数：只写一个类型进行占位，调用时才传入值。 int func(int a, int)；占位也可以有默认值。 int func(int a, int = 1)

引用

  • 目的是起别名，增加访问方式。语法 类型 &别名=原名
  • 引用必须要初始化，且一旦初始化以后，就不可以指向其他变量，因为引用本质上是一个常量指针。
  • 数组引用写法 int arr[100]; int(&pArr)[100]=arr;
  • 引用注意事项，引用必须引用一块儿合法的内存空间，例如不能出现 int& a =10,因为10是存在于常量缓冲区的；不要返回局部变量的引用；当函数的返回值是引用的时候，那么真个函数是可以当作左值存在的。
  • 常量的引用是可以的：const int& ref=100，加了const以后相当于执行了 int tmp=100; int& ref =tmp;

函数传参方式

  • 值传递
  • 引用传递
  • 地址传递

0x02 类相关

封装

  • 产生原因因为C语言中的属性和行为是混合在一起的，不利于表述。而C++中的封装将属性和行为作为一个整体，来表现生活中的一些事物，并提供权限访问，利于表述。
  • 访问权限分为三种：public(公共)；protected(保护)；private(私有)
  • class的默认权限是私有权限，而struct的默认权限却是公共权限。
  • 尽量将成员属性设置为私有，并提供函数进行访问，这样利用做内容上的控制和校验。

对象模型

  • 类中的成员变量和成员函数是分开存储的
  • 只有非静态成员变量是属于类对象 的
  • 空对象的 sizeof =1，起标识作用。

构造函数和析构函数

  • 构造函数的调用规则，编译器会给一个类提供三个函数：默认构造(空实现)，析构函数(空实现)，拷贝构造(值拷贝)。当我们提供了有参构造函数后，编译器将不会提供默认构造。同理如果我们提供了拷贝构造寒素，那么编译器也将不会提供。
  • 拷贝构造的执行时机有三种(时刻要注意)：用已经创建好的对象来初始化新的对象；以值传递方式给函数传递参数；以值的形式放回函数局部对象。
  • 构造函数写法 Person() 可以有参数，可以发生重载，不可以有返回值，和类名保持相同，编译器会调用默认构造函数。
  • 析构函数写法 ~Person() 不可以有参数，不可以重载，不可以有返回值，当对象释放时编译器也会默认调用。
  • 构造函数调用方式，包括括号法 Person p(100); 显式调用法 Person p=Person(100)；隐式调用法 Person p=100;
  • 构造函数按照参数划分为有参构造和无参构造；按照类型划分为默认构造和拷贝构造。
  • 如果函数中出现了Person(100)，代表匿名对象，当前执行完立即释放。
  • 初始化列表：可以利用初始化列表对属性初始化，写法Person(int age):m_Age(age)；
  • 类对象作为类中成员：其他类对象作为本类成员时先调用其他对象构造，再调用自身构造，析构和构造顺序相反。
  • 深拷贝和浅拷贝的问题，因为编译器提供的拷贝构造函数默认是浅拷贝，所以在析构的时候会引发堆区内存重复释放问题，通过重载operator=运算符来实现深拷贝解决。
  • new和delete的区别：malloc和free属于库函数，而new和delete属于运算符；malloc不会调用构造函数，new会调用构造函数；malloc返回void*，c++下要强转，new返回创建对象的指针；不要用void*去接受new出来的对象，因为利用void*无法调用析构函数；
  • 数组的new创建方式 Person* parr=new Person[100]; 释放的时候 delete[] parr;

this指针

  • this指针指向被调用的成员函数所属的对象
  • this指针隐式的加在每个成员函数中
  • *this代表的就是对象本体
  • this指针还可以解决名称冲突的问题
  • 如果函数中没有用到this指针，可以利用空指针调用函数，如果用到了则必须判断，否则当访问数据时，由于是NULL，会引发异常中断程序。

继承

  • 优点是减少重复代码，提高代码的复用性。
  • 写法 class 子类:继承方式 父类 class Son:public Base
  • 继承方式分为公共继承(权限完全保持父类的)，保护继承(公共权限将变为保护权限)，私有继承(全部变为私有权限)。
  • 继承的对象模型：父类中的所有属性包括私有其实子类都已经继承下去了，只不过编译器隐藏了访问不到。可以利用VS的开发人员工具查看 cd到文件目录，然后输入指令: cl /d1 reportSingleClassLayout类名 文件名
  • 继承中的构造和析构执行顺序：先调用父类的构造函数，再调用其他成员的构造函数，最后调用自身构造，析构顺序与此相反。
  • 编译器提供的默认构造，默认析构，拷贝构造，以及operator=是不会被子类继承下去的。
  • 当子类中出现了同名的函数时，可以利用作用域运算符::调用父类同名函数。原因时一旦子类中重新定义了父类同名函数时，那么父类所有的重载函数都将被隐藏，只有通过作用域来访问。
  • 多继承： class 子类：继承方式 父类1，继承方式 父类2；当多维继承的父类中存在同名成员函数时，可以通过作用域::访问。
  • 菱形继承问题以及解决：发生条件当两个类有公共的父类和公共的子类的时候，这种继承方式称为菱形继承。菱形继承存在弊端：会导致数据存有两份，浪费资源(可以通过上述提到VS命令查看对象布局)； 解决方式：通过虚继承来解决，即加virtual关键字。原理大概是当发生虚继承时，两个子类继承了一个vbptr的指针，虚基类指针指向的是一个虚函数表vbtable，虚函数表中记录了数据的偏移量，通过偏移量可以找到唯一的一份数据并访问。

多态

  • 原理：当父类写了虚函数以后，类的内部结构发生了变化，多了一个vfptr的虚函数指针，该指针指向虚函数表 vftable。虚函数表中记录着虚函数的入口地址，当父类指针指向子类对象的时候，发生多态，调用的时候会从虚函数表中找到入口地址并调用。
  • 虚函数关键字 virtual。
  • 纯虚函数 virtual void show()=0，没有实现，又称为抽象类，无法被实例化，且子类必须重写纯虚函数，否则子类也是一个抽象类。
  • 纯虚析构函数 virtual ~Person()=0; |Person::~Person(){}； 既需要声明，也需要实现。 如果一个类中有了纯虚析构函数，那么该类也是一个抽象类。
  • 静态联编和动态联编：静态多态包括：函数重载，运算符重载，编译阶段绑定地址，成为静态联编；动态多态指的是父类指针指向子类对象，首先要有继承关系，然后父类中有虚函数存在，子类中重写父函数才行，运行阶段绑定地址，又称动态联编。

类型转换

  • 静态类型转换 static_cast<目标类型>(变量/对象)，允许内置数据类型和父子之间的指针或引用的转换。
  • 动态类型转换 dynamic_cast<目标类型>(变量/对象)，不允许内置数据类型的转换，只允许父子之间的指针或引用转换。由于子类实际上在内存布局上是对父类的扩充，所以父转子是安全的，而子转父可能会出现数据越界的情况，而如果发生了多态，则转换永远是安全的。
  • 常量转换 const_cast<目标类型>(变量/对象)，只允许指针或引用的转换。
  • 重新解释类型转换：reinterpret_cast<目标类型>(变量/对象)，最不安全的一种转换，什么都可以转，无法保证有效性，所以尽量不使用。

0x03 模板相关

函数模板

  • 优势：将类型参数化，实现泛型编程
  • 语法 template<typename T> void func(T a); T代表一个通用类型
  • 使用的两种方式：自动推导 func(1,3.14)；显示指定类型 func<int,double>(1,3.14)

函数模板和普通函数对比关系

  • 如果函数模板和普通函数都可以调用，那么优先调用普通函数，因为函数模板要发生类型推导，普通函数直接调用就可以啦。
  • 函数模板也可以发生重载。
  • 如果函数模板能产生更好的匹配，则会优先调用函数模板。
  • 对于函数模板如果采用自动推导方式，无法完成隐式的类型转换，而普通函数而已。

函数模板的实现机制

  • 编译器会执行两次编译，第一次在声明的地方对模板代码进行编译，第二次在调用的地方将类型替换成确定的类型后对代码进行再次编译。
  • 由编译器产生的具体的函数，称为模板函数。

类模板

  • 语法 template<class T> class Person{public: T a;}
  • 类模板不可以使用自动推导类型的方式，只能用显示指定的方式调用
  • 类模板可以指定默认参数
  • 类模板中的函数不是一开始创建的，而是在运行阶段确定了T的数据类型后才创建的。
  • 当类模板遇到继承的时候，必须先指定出父类的类型，才可以给子类分配内存。
  • 当类模板的文件分开编写的时候，如果在程序中只引入了.h文件，那么由于类模板中的函数是在确定了类型后产生的，所以会引发没有可以解决的外部符号的链接错误，解决方式有两种，将.h文件和.cpp都包含进行；或者写在一个文件里且将.h改为.hpp不易混淆。
  • 当类模板遇到友元的时候，既可以在类内实现，也可以在类内声明，类外实现。

0x04 异常和文件读写

异常

  • 关键字 try throw catch
  • 可以将可能出现异常的代码放到try{}代码块中，当出现异常时，利用throw抛出异常，然后利用catch捕获异常。 catch(类型)捕获什么类型的异常。
  • 栈解旋：从try代码开始，到throw抛出异常前，栈上所有的对象都将被清空释放，这种现象称为栈解旋。
  • 异常的接口声明，void func() throw(int) 代表只能抛出int类型的异常。 throw()则代表不允许抛出异常。
  • 可以自定义自己的异常类，甚至时自定义异常基类，比如VS底层应该有一套class BaseException基类，然后class NullPointerException:BaseException(空指针异常)，OutofRangeException:BaseException(数据越界)等等各个异常子类。

标准输入流

  • 无论是输入流还是输出流，都不是直接输入到屏幕，或者从键盘直接输入，中间都存在一个缓冲区，我们操作的是从缓冲区中读取和写入数据的。
  • cin.get()读取一个字符 换行符停留在缓冲区。
  • cin.getline()获取字符串，获取不到换行符，换行符也不在缓冲区，被直接清除了。
  • cin.peek() 获取到第一个，但是不拿出来，只是查看数据而已。
  • cin.putback() 将字符放回原位置。
  • cin.ignore() 忽略一个字符

标准输出流

  • cout.width(10) 指定宽度为10
  • cout.fill('*') 用*填充
  • cout.set(ios::left)设置左对齐

文件读写

  • 头文件是 #include<fstream>
  • 读文件： ifstream ifs(文件路径,打开方式) ，判断文件是否打开 ifs.is_open()，开始读操作ifs>>""，关闭读操作close。
  • 写文件：ofstream ofs(文件路径，打开方式)，判断文件是否打开ofs.is_open()，开始写操作ofs<<""，关闭写操作close。

STL基本概念：

  • 六大组件：容器，算法，迭代器，仿函数，适配器，空间配置器
  • STL优点：内置在C++编译器中，不用额外安装其他内容；高复用，可移植；无需剖析底层算法，熟练使用就可以。
  • 容器和算法通过迭代器无缝连接，迭代器是桥梁
  • 容器大致可划分为：序列式容器(vector)和关联式容器(map,键值对存储)
  • 算法大致可划分为：非质变算法(查找，不改变原来数据)和质变算法(改变原来的数据)
  • 容器可以存放自定义的数据类型和自定义数据类型的指针，而不仅仅是int,float,string基本数据类型
  • 容器可以嵌套容器

string：基本api简单列举

vector

  • 单向数组
  • 动态数组，自动扩展内存，但这里的自动扩展不是在原来的基础上续增开辟，而是会创建一块儿新的独立的空间，然后把数据拷贝过去，再删除掉原来的内存空间。
  • 简单api：插入inset；删除erase；清空clear；尾插pushback；尾删popback；是否为空empty；重置大小resize；交换swap；大小size；逆序遍历reverse_iterator

deque

  • 双向数组
  • 可以对头部进行插入和删除操作，内部有中控器控制数据(底层实现原理)
  • 简单api：插入inset；删除erase；清空clear；头部插入pushfront；头部删除pop_front；尾插push_back；尾删pop_back；访问第一个元素front；访问最后一个元素back；是否为空empty；重置大小resize；交换swap；大小size；

stack

  • 先进后出
  • 简单api：入栈push；出栈pop；栈顶top；是否为空empty；栈大小size

queue

  • 先进先出
  • 简单api：入队push；出队pop；对头元素front；队尾元素back；是否为空empty；大小size

list

  • 双向循环列表
  • 简单api：大小size；是否为空empty；反转reverse；排序sort，自定义数据类型需要指定排序规则，告诉编译器按照什么规则排序；调用remove需要重载==

set

  • 关联式容器，key就是value，在插入的时候就会自动排好序
  • 简单api：插入insert；大小size；是否为空empty；查找find；统计count；返回第一个>=的元素lowerbound(keyEle)；返回第一个>的元素upperbound(keyEle)；equal_range(keyEle)返回与keyEle相等的上下两个迭代器
  • 对于自定义数据类型，通常是需要指定排序规则
  • multi_set可以添加重复元素

map

  • 关联式容器，插入的时候就已经默认从小到大排好序了
  • 利用仿函数，指定排序规则
  • 简单api：插入insert(pair<type,type>(a,b)), insert(make_pair(a,b)),insert(map<type,type>::map_value)；查找find；返回第一个>=的元素lowerbound(keyEle)；返回第一个>的元素upperbound(keyEle)；equal_range(keyEle)返回与keyEle相等的上下两个迭代器
